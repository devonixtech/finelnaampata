# 6\. Technical & Organisational Security Measures \(TOMs\)

__Security Domain__

__Standard Implemented by NAAMPATA__

Encryption at Rest

AES\-256 encryption for all Personal Data stored in production databases and AWS S3 file storage systems\.

Encryption in Transit

TLS 1\.2 minimum enforced on all endpoints, APIs, and data transfer channels\. TLS 1\.0 and TLS 1\.1 explicitly disabled\.

Access Control Architecture

Role\-Based Access Control \(RBAC\) with the principle of least privilege; mandatory two\-factor authentication \(2FA\) for all admin and privileged accounts; session timeout controls\.

Admin Panel Security

Accessible exclusively from pre\-approved, whitelisted IP addresses\. Session expiry enforced on all privileged access sessions\.

Network & Perimeter Security

Web Application Firewall \(WAF\) with OWASP Core Rule Set on all public endpoints; network segmentation between production and development environments; DDoS mitigation\.

Vulnerability & Patch Management

Annual third\-party penetration testing\. Critical security findings remediated within 5 business days\. High\-severity findings within 30 business days\. Patch management process documented\.

Audit Logging

Immutable, tamper\-evident audit logs for all administrative and privileged actions, retained for a minimum of 24 months, with real\-time alerting for anomalous access patterns\.

Physical & Cloud Security

All production data processing conducted on AWS cloud infrastructure\. AWS holds SOC 2 Type II and ISO 27001 certification\. No NAAMPATA\-managed physical servers in production\. Physical security managed by AWS per their published security standards\.

Backup & Disaster Recovery

Daily automated encrypted backups with quarterly tested restoration procedures\. Defined and documented Recovery Time Objective \(RTO\) and Recovery Point Objective \(RPO\)\.

Staff Training & Awareness

Annual mandatory data protection and security awareness training for all staff with access to Personal Data\. Incident response training for security\-relevant roles\.

Sub\-Processor Security

All Sub\-Processors required by contract to maintain security measures at least equivalent to those in this Section\. Security posture verified before engagement and reviewed periodically\.

Password & Credential Management

All passwords stored as bcrypt one\-way cryptographic hashes\. Privileged access credentials rotated on a defined schedule\. No plaintext credential storage under any circumstance\.

These measures are reviewed and updated at a minimum annually and upon any material change to the processing environment, threat landscape, or applicable regulatory requirements\. NAAMPATA may satisfy audit rights through provision of current SOC 2 Type II or ISO 27001 certification reports from accredited third\-party auditors as a proportionate alternative to full on\-site audit, per Section 11\.
